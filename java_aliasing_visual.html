Box a = new Box(7); // a points to Box object
Box b = a; // b gets SAME reference as a
b.value = 200; // changes the ONE object
System.out.println(a.value); // 200 (same object!)
System.out.println(b.value); // 200 (same object!)
Why `a.value` prints 200:

1. `Box b = a` copies the reference from `a` to `b` - it doesn't create a new Box object
2. Both `a` and `b` contain the same memory address (@1A2B3C)
3. There is only ONE Box object in memory, but two variables pointing to it
4. `b.value = 200` changes the value field of that one object
5. `a.value` accesses the same object, so it sees the changed value: 200
